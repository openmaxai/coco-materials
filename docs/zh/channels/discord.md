# Discord — 使用技巧

在 Discord 上使用 OpenMax AI 员工，适用于开发者社区、支持服务器和团队协作。

## 基本使用

### 私信
直接向机器人发送消息，私聊中无需 @mention——打开与机器人的私信对话即可开始。

### 服务器频道与线程
在任意服务器频道或线程中 @mention 机器人：
```
@OpenMax AI Employee 总结一下最新的发布说明
```

## 适用场景

### 开发者与社区服务器
在开发者或社区服务器中部署，提供即时、全天候的帮助：
- 回答配置和故障排查问题
- 总结长讨论线程
- 按需撰写公告和更新日志

### 社区支持
在支持频道中：
- @mention 机器人对收到的问题进行分流
- 为国际成员翻译消息
- 无需离开 Discord 即可查阅文档和答案

### 团队协作
Discord 适合分布式团队和兴趣项目：
- 在专用频道中保持项目讨论和任务有序
- 使用线程让繁忙的频道保持整洁
- 分享文件和代码片段以便快速分析

## 进阶技巧

**智能模式**：在服务器频道中启用智能模式——机器人接收所有消息并根据上下文决定何时响应，无需每次都 @mention。

**文件与图片共享**：直接在聊天中拖入图片、PDF 等文件。机器人可以读取、分析并回复附件内容。

**表情回复**：机器人可以添加表情回复来确认并响应消息。

**线程**：机器人支持在线程内回复，在繁忙的频道中保持对话有序。

**Owner（管理员）**：第一个给机器人发送私信的用户将成为 Owner，始终拥有完整访问权限，不受访问设置限制。

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| 机器人离线 | 确认 Bot Token 正确，并已在 Discord 开发者门户中启用 Message Content Intent |
| 服务器频道中无回复 | 确认机器人已被邀请进服务器、在该频道中拥有查看和发送消息的权限，并已被 @mention |
| 机器人收到空消息或无消息文本 | 在 Discord 开发者门户的 Bot 标签页中启用 **Message Content Intent** |
| 无法给机器人发私信 | 私信无需服务器邀请——打开与机器人的私信对话并发送任意消息即可 |
| 想要断开连接 | 在员工详情页的 Discord 卡片上点击 **断开连接** 按钮 |
