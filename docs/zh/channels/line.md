# LINE — 使用技巧

在 LINE 上使用 OpenMax AI 员工，适用于个人助理、面向客户的支持和团队聊天——尤其在 LINE 作为主力即时通讯应用的日本、台湾和泰国。

> 第一次使用？请先查看 [LINE 部署指南](../getting-started/channel-deployment#line) 连接你的官方账号。

## 基本使用

### 私信
将官方账号加为好友，直接发送任意消息即可——无需特殊命令。第一个给该账号发私信的用户会成为 **Owner（管理员）**，拥有完整访问权限。

### 群聊
将官方账号邀请进 LINE 群组或多人聊天。要允许其加入群组，需在 LINE 官方账号管理后台（设置 → 账号设置）中开启 **"允许官方账号加入群聊"**。群组访问默认按白名单控制。

## 适用场景

### 面向客户的支持
LINE 非常适合服务日本、台湾、泰国客户的企业：
- 全天候即时回答常见问题和产品咨询
- 对收到的请求进行分流和路由
- 为国际客户翻译消息

### 个人助理
作为随身的全天候个人助理：
- 通过聊天记录笔记、提醒和待办
- 总结转发的文章、文档和语音备忘
- 无需离开 LINE 即可查询信息

### 团队与社区
对于活跃在 LINE 上的团队和社区：
- 在群聊中推进讨论
- 分享文件和图片以便快速分析
- 按需查阅答案和文档

## 进阶技巧

**媒体收发**：发送图片、视频、语音消息和文件——AI 员工能读取它们（语音会自动转写），也能回发图片、视频和音频。

**语音消息**：录制一条语音消息，AI 员工会转写并据此处理，然后以文字回复。

**Owner（管理员）**：第一个给官方账号发私信的用户将成为 Owner，始终拥有完整访问权限，不受访问设置限制。

**访问控制**：默认只有 Owner 能私信该账号，且群组需显式允许。可调整私信和群组策略，向团队或公众开放。

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| 机器人无响应 | 确认 LINE Developers Console 中已开启 **Use webhook**，且保存的 Webhook URL 与 LINE 卡片上显示的完全一致（包含 `/line/webhook`） |
| 回复来自 LINE 的自动回复而非 AI | 在官方账号管理后台 → 回复设置中关闭 **自动回复消息**（保持 **Webhooks** 开启） |
| 已连接但收不到消息 | 确认已将官方账号加为好友，且回复设置中已开启 **Webhooks** |
| 群聊消息被忽略 | 开启"允许官方账号加入群聊"，并确认该群组在你的访问设置中已被允许 |
| 想要断开连接 | 在员工详情页的 LINE 卡片上点击 **断开连接** 按钮 |
